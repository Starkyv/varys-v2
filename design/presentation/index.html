The problem

Your screens break in ways code tests never see.

It still ships green

A shifted button, a vanished logo, a layout that collapsed on one customer — the unit tests pass anyway.

Eyeballs don't scale

Manually checking every screen, every release, across every environment is the work nobody has time for.

Brittle to write

Hand-written end-to-end tests are slow to author and snap the moment the UI moves an inch.

In one line

Varys records a journey through your app, replays it after every change, and shows you exactly what looks different.

It's visual-regression testing — automated. The screen is the assertion.

Replay · the run

Varys re-walks the journey, server-side.

A run is one replay of a saved test against one environment. A fresh cloud browser logs in clean, repeats every step, and snaps a photo at each checkpoint.

Identical every time

Same browser build, same fonts, same window size, animations frozen — so a difference is a real difference, never noise.

Smart waiting

It waits for the page to settle before each shot — no flaky "too early" captures.

Baseline

The "known-good" picture — one per environment.

A baseline is the reference image every future run is compared to. Approve a screen once, and it becomes the standard.

Why Varys makes the baseline — not your laptop

Your machine and the cloud renderer draw fonts slightly differently. So the very first run generates the golden images itself; you just eyeball and approve them once. Recording ≠ baseline.

Robustness

Tests that bend instead of break.

Most tests remember an element by a single brittle selector. Varys remembers it by many signals at once — a fingerprint:

rolevisible textaccessible namenearby labels tagattributesancestor structurepositionbounding box

Self-heal

If the page shifts, Varys re-finds the element from the surviving signals and carries on — flagging it "low-confidence, review" rather than failing outright.

Honest failure

If nothing matches with confidence, it stops and surfaces the element for a quick repair — never a silent wrong click.

Environments & variables

Record once. Run everywhere.

Dev, demo, and every customer is just another environment. Varys swaps in the right address, login, and data automatically.

Secrets stay safe — encrypted, used only inside the runner, and scrubbed from every screenshot, video, and log.
