title: Environments

---

An **environment** lets one test run in different places — staging, production, your local
machine — without rewriting it. When you run a test, you pick which environment to run it
against.

Each environment holds:

<Cards>
  <Card title="Base URL" description="The web address this environment points at — the one thing that differs between staging, production, and local." />
  <Card title="Cookies" description="Set automatically before each run — handy for skipping login, for example." />
  <Card title="Local storage" description="Key/value entries set in the browser before each run — another way to skip login when your app keeps its session there." />
</Cards>

## Why it matters

A test records its steps once. The only thing that changes between places is the **base URL**, so
that's all the environment supplies (along with any cookies or local storage needed to start
signed in). Everything else you typed while recording stays in the test as-is. Each environment
also keeps its own approved baselines, since staging and production legitimately look different.

<Callout type="info">
  Pick the environment when you run a test — the run uses its base URL, cookies, and local storage.
</Callout>
