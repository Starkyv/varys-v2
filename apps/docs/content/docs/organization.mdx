---
title: Folders & tags
description: Keep your tests organized.
---

As you build up tests, two tools keep them tidy.

## Folders

A **folder** is a test's home — each test lives in exactly one. Create folders for areas of your
app (say, *Dashboard* or *Checkout*), and drag a test onto a folder to file it. Tests you haven't
filed show up under **Unfiled**.

Deleting a folder doesn't delete its tests — they just move back to Unfiled.

## Tags

A **tag** is a label you can put on any test, and a test can have several. Unlike folders, tags
cut across everything — so you can group tests by release, feature, or anything else (for
example `release:5.0` or `feature:dashboard`) no matter which folder they live in.

<Callout>
  Think of it this way: folders are *where a test lives*; tags are *how you slice across them*.
</Callout>
