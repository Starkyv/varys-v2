---
title: Scheduling
description: Give a test a regular cadence.
---

You can put a test on a schedule so it's meant to run on its own — **hourly, daily, weekly, or a
custom cadence** — in the timezone you choose and against a specific
[environment](/docs/environments). Varys shows you when it would next run.

<Callout type="warn">
  Automatic running isn't switched on yet. For now a schedule is saved and the next-run time is
  shown, but runs still need to be started manually. Once it's live, scheduled tests will run on
  their own with no other change.
</Callout>
