---
title: Review queue
description: Review the tests Claude created before they go live.
---

Every test Claude creates waits in the **review queue** until you approve it — so nothing Claude
makes goes live without you looking at it first.

Open the **Review queue** in the app. Drafts are listed on the left; selecting one shows, on the
right, what Claude was asked to do and preview screenshots of what the test checks.

## What you can do with a draft

<Cards>
  <Card title="Rename" description="Give the test a clearer name." />
  <Card title="Open in the editor" description="Tweak it — adjust waits or sensitivity, or remove a step." />
  <Card title="Run a preview" description="Try it against a test environment to see what it captures." />
  <Card title="Promote" description="File it into a folder with tags and make it live." />
  <Card title="Discard" description="Throw away a draft you don't want." />
</Cards>

<Steps>

<Step>
### Look it over
Check what Claude was asked to do and the preview screenshots. If a draft doesn't actually check
anything, the queue warns you.
</Step>

<Step>
### Adjust if needed
Open it in the editor to fine-tune, or run a preview to see it in action.
</Step>

<Step>
### Promote it
When you're happy, **promote** it — pick a folder and tags, and it joins your live tests.
</Step>

</Steps>

<Callout>
  Promoting a test makes it live. Approving its baseline screenshots is a separate step you do the
  first time it runs — so you both file the test *and* sign off on what it looks like.
</Callout>
